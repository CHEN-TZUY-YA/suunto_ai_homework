# combined_temperature_test_data.csv 校验记录

## 校验结论

- `combined_temperature_test_data.csv` 的数值列和原始 Excel 清洗逻辑逐行一致；按现有 `temperature_excel_integration.py` 重建后，107 行、9 列没有发现数值或列映射不一致。
- 原始 Excel 中可解析的记录共 108 行：`场景123登记表` 75 行，`全天场景体温记录表` 33 行。现有整合逻辑丢弃了 1 行：邱晓健 `重新佩戴`，时间 `2026-03-31 22:12:00`，原因是小米和 CORE 核心温为空/不可用。
- 主要问题在 `Time` 日期：`场景123登记表` 只有时分秒，整合脚本把这 75 行统一补成 `2026-03-31`。这不是实际实验日期，会影响按日期排序、相邻变化量、跨天趋势等分析。

## 对统计报告的影响

- MAE、RMSE、偏差、最高误差等逐点误差统计不受日期问题影响。
- 相邻趋势、delta 相关、昼夜/跨日排序会受影响，因此已在 `temperature_statistical_report.md` 中改用 `Time_Corrected`，并按同一受试者、同一大类、同一主题会话内计算相邻变化。
- 原始 `combined_temperature_test_data.csv` 未覆盖；校正后的处理表另存为 `processed_temperature_data_with_corrected_dates.csv`。

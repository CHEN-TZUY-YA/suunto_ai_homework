# Temperature Wearable Report

This repository contains the summarized outputs for a temperature wearable analysis project.

## Contents

- `polish.md`  
  A concise record of what was changed or added in this project, including:
  - data validation and timestamp correction work;
  - statistical analysis scripts and outputs;
  - generated reports and charts;
  - the static web report;
  - Codex features/tools used during the work.

- `analysis_output/`  
  The generated analysis outputs, including summary CSV files, Markdown reports, plots, corrected processed data, and the static web report.

- `analysis_output/web_report/index.html`  
  A summary web page for product/algorithm review. It explains:
  - main use cases for wearable temperature signals;
  - user needs for core temperature, skin temperature, and DPG;
  - how CORE 1, CORE 2, rectal capsule reference, mercury thermometer, and Xiaomi temperature compare;
  - key statistics from the local Xiaomi / mercury / CORE test data;
  - algorithm-readiness judgments for menstrual cycle, sleep onset, circadian rhythm, recovery, and heat-load scenarios.

## Data and Source Notes

The web report includes a small amount of local test data from this project.

The CORE 1 / CORE 2 / rectal temperature capsule discussion is based on the user-provided summary of a YouTube review by The Quantified Scientist:
